أخطاء التثبيت والتجهيز الأولي

حرج Python غير موجود أو غير معروف
'python' is not recognized as an internal or external command, operable program or batch file.
الحل:
  1. حمّل Python 3.12 من python.org
  2. عند التثبيت ضع علامة "Add Python to PATH"
  3. أعد فتح Terminal/PowerShell
  4. تأكد: python --version
PowerShell # للتحقق من إصدار Python python --version # للتحقق من مسار Python where.exe python
حرج Django غير مثبّت (ModuleNotFoundError: No module named 'django')
ModuleNotFoundError: No module named 'django'
الحل:
  1. تأكد من تفعيل البيئة الافتراضية أولاً
  2. ثم ثبّت المتطلبات
PowerShell # تفعيل البيئة الافتراضية venv\Scripts\activate # تثبيت المتطلبات pip install -r requirements.txt # للتحقق pip list | findstr Django
تنبيه
تأكد أنك تستخدم Python الصحيح داخل البيئة الافتراضية وليس Python النظام. إذا ظهر خطأ أثناء pip install، جرّب تشغيل Terminal كمسؤول (Run as Administrator).
حرج python-dotenv غير مثبّت (ModuleNotFoundError: No module named 'dotenv')
ModuleNotFoundError: No module named 'dotenv'
الحل:

المكتبة مُضاف في requirements.txt لكنها قد لا تكون مثبّتة يدويًا في البيئة الافتراضية:

PowerShell venv\Scripts\activate pip install python-dotenv
عالية البيئة الافتراضية (venv) غير موجودة
The system cannot find the path specified: 'venv\\Scripts\\python.exe'
الحل — إنشاء البيئة الافتراضية заново:
PowerShell # إنشاء البيئة الافتراضية python -m venv venv # تفعيلها venv\Scripts\activate # تثبيت المتطلبات pip install -r requirements.txt # تثبيت python-dotenv pip install python-dotenv
عالية فشل تثبيت مكتبة أثناء pip install
ERROR: Could not build wheels for psycopg2
الحل:
  1. psycopg2-binary: استخدم النسخة البديلة: pip install psycopg2-binary
  2. cairosvg: لا يلزم — التقرير يدعم SVG بشكل افتراضي
  3. أخطاء build tools: حمّل Microsoft Visual C++ Build Tools
  4. أو جرّب: pip install --upgrade pip setuptools wheel ثم أعد المحاولة
حرج DJANGO_SECRET_KEY مفقود في وضع الإنتاج
RuntimeError: DJANGO_SECRET_KEY environment variable is required in production.
الحل:

في وضع الإنتاج (DEBUG=False) يجب تعيين المفتاح إما في ملف .env أو كمتغير بيئة:

.env # أضف مفتاح سري ثابت DJANGO_SECRET_KEY=your-super-secret-key-here-change-this DJANGO_DEBUG=false
حماية
لا تشارك المفتاح السري أبدًا ولا ترفعه لـ Git. في وضع التطوير (DEBUG=true)، Django يولّد مفتاحًا مؤقتًا تلقائيًا.

أخطاء تشغيل السيرفر

حرج البورت مستخدم بالفعل (Address already in use)
OSError: [WinError 10048] An attempt was made to access a socket in a way forbidden by its access permissions 0.0.0.0:8001
الحل:
  1. اكتشف العملية التي تستخدم البورت
  2. أوقفها أو غيّر البورت في network.conf
PowerShell # معرفة العملية التي تستخدم البورت 8001 netstat -ano | findstr :8001 # ستظهر نتيجة مثل: # TCP 0.0.0.0:8001 0.0.0.0:0 LISTENING 6576 # أوقف العملية بالمعرف (PID) taskkill /PID 6576 /F # أو غيّر البورت في network.conf # PORT = 8002
متوسطة تنبيه StatReloader: module 'watchdog' not found
WARNING: StatReloader: module 'watchdog' is not installed, falling back to PollingReloader
الحل:

هذا تحذير فقط ولا يمنع عمل السيرفر. يمكنك تثبيت watchdog لتحسين أداء المراقبة:

PowerShell pip install watchdog
عالية whitenoise غير مثبّت — ملفات ثابتة لا تُخدم
ModuleNotFoundError: No module named 'whitenoise'
الحل:

whitenoise مطلوب لخدمة الملفات الثابتة في وضع الإنتاج ومهيأ في middleware:

PowerShell pip install whitenoise
عالية رفض الوصول لملف manage.py
PermissionError: [Errno 13] Permission denied: 'manage.py'
الحل:
  1. تأكد أنك تشغل PowerShell كمسؤول (Run as Administrator)
  2. أو غيّر صلاحيات المجلد
  3. أو افتح المجلد في محرر كود وشغّله من داخله

أخطاء ملف .env

عالية ملف .env لا يُحمّل
المشكلة
ملف .env لا يُحمّل تلقائيًا في Python — يجب استدعاؤه يدويًا عبر python-dotenv.
الحل:

تأكد أن كل ملفات التشغيل تحتوي load_dotenv() في الأعلى:

Python from dotenv import load_dotenv import os load_dotenv(os.path.join(os.path.dirname(__file__), '.env'))
ملاحظة
ملفات التشغيل المدعومة: run_server.py, run_waitress.py, run_multi.py. ملف install.bat يضبط المتغيرات مباشرة في cmd.
عالية البورت أو IP غير صحيح في network.conf
Server starts on port 8012 but browser opens http://127.0.0.1:8001
الحل:

تأكد من توافق البورت في:

network.conf # يجب أن يكون البورت موحّدًا INTERNAL_IP = 192.168.1.31 EXTERNAL_IP = 196.218.24.45 PORT = 8001 # ← نفس البورت في المتصفح
.env BIND_PORT=8001 # ← يُفضّل تعيينه هنا أيضًا
كيف تتحقق؟
netstat -ano | findstr :8001 — يجب أن تظهر العملية تستمع على البورت الصحيح.

أخطاء قاعدة البيانات

حرج قاعدة البيانات مقفلة (database is locked)
django.db.utils.OperationalError: database is locked
الحل:

النظام يستخدم SQLite مع إعدادات WAL و busy_timeout. إذا ظهر هذا الخطأ:

  1. تأكد أن عملية أخرى لا تكتب لنفس القاعدة
  2. أعد تشغيل السيرفر
  3. احذف ملفات WAL إذا تلفت: db.sqlite3-wal و db.sqlite3-shm
  4. للإنتاج الكثيف: انتقل إلى PostgreSQL
PowerShell # أوقف السيرفر أولاً، ثم احذف ملفات WAL Remove-Item db.sqlite3-wal -ErrorAction SilentlyContinue Remove-Item db.sqlite3-shm -ErrorAction SilentlyContinue # ثم أعد التشغيل venv\Scripts\python run_server.py
انتبه
لا تحذف ملف db.sqlite3 نفسه — هذا سيحذف كل البيانات!
حرج الجدول غير موجود في قاعدة البيانات
django.db.utils.OperationalError: no such table: accounts_account
الحل:

هذا يعني أن الهجرات (migrations) لم تُطبَّق:

PowerShell # تطبيق جميع الهجرات venv\Scripts\python manage.py migrate # أو إعادة الهجرة من الصفر (حذف وإعادة بناء) venv\Scripts\python manage.py migrate --fake venv\Scripts\python manage.py migrate
عالية ملف قاعدة البيانات غير موجود (db.sqlite3)
OperationalError: unable to open database file
الحل:
  1. تأكد أنك في المجلد الصحيح (D:\accounting_system)
  2. أو أعد إنشاء قاعدة البيانات:
PowerShell # إنشاء الهجرات venv\Scripts\python manage.py makemigrations # تطبيقها لإنشاء قاعدة البيانات venv\Scripts\python manage.py migrate # إنشاء مستخدم مسؤول venv\Scripts\python manage.py createsuperuser
عالية تعارض في الفريد (IntegrityError)
django.db.utils.IntegrityError: UNIQUE constraint failed: accounts_account.code
الحل:
  1. هذا يعني محاولة إدخال سجل بـ code مكرر
  2. في النظام، الكود (code) فريد لكل نوع حساب / منتج / إلخ
  3. غيّر الكود إلى قيمة فريدة
  4. أو تحقق من عدم وجود سجل مكرر عبر واجهة الإدارة

أخطاء الهجرة (Migrations)

عالية dependency issues or circular reference في الهجرات
django.db.migrations.exceptions.InconsistentMigrationHistory LookupError: No app labeled 'concrete_production'
الحل:

الترتيب مهم — بعض التطبيقات تعتمد على غيرها:

PowerShell # ترتيب الهجرة الصحيح: venv\Scripts\python manage.py migrate accounts venv\Scripts\python manage.py migrate purchases venv\Scripts\python manage.py migrate sales venv\Scripts\python manage.py migrate treasury venv\Scripts\python manage.py migrate concrete_production venv\Scripts\python manage.py migrate contractors venv\Scripts\python manage.py migrate
عالية العمود موجود بالفعل في الهجرة
django.db.utils.OperationalError: column "field_name" already exists
الحل:
  1. هذا يحدث عند تعديل model ثم تشغيل makemigrations مرتين
  2. أوقف السيرفر
  3. احذف أحدث ملف هجرة في التطبيق المعني
  4. أعد إنشاء الهجرة:
PowerShell venv\Scripts\python manage.py makemigrations <app_name> venv\Scripts\python manage.py migrate
عالية الحقل لا يملك قيمة افتراضية
django.db.utils.IntegrityError: NOT NULL constraint failed
الحل:

أضفت حقل جديد بدون قيمة افتراضية في model مع وجود بيانات سابقة:

  1. أضف null=True, blank=True للحقل في model
  2. أو أضف default=0 أو default=''
  3. أو عدّل الهجرة يدويًا لإضافة قيمة مؤقتة

أخطاء تسجيل الدخول والمصادقة

حرج فشل تسجيل الدخول — كلمة المرور أو اسم المستخدم خاطئ
الأعراض
تظهر صفحة تسجيل الدخول مرة أخرى بدون رسالة خطأ واضحة.
الحل:
  1. المستخدم الافتراضي: admin / admin123
  2. إذا نسيت كلمة المرور:
PowerShell # إعادة تعيين كلمة مرور المستخدم admin venv\Scripts\python manage.py shell -c " from django.contrib.auth.models import User u = User.objects.get(username='admin') u.set_password('admin123') u.save() print('Password reset successfully!') "
عالية Session data corrupted — الجلسة تالفة
WARNING: Session data corrupted
الحل:
  1. هذه المشكلة تظهر عند تغيير SECRET_KEY مع وجود جلسات قديمة
  2. امسح ملفات الكوكيز من المتصفح
  3. أو احذف جلسات قاعدة البيانات:
PowerShell venv\Scripts\python manage.py shell -c " from django.contrib.sessions.models import Session Session.objects.all().delete() print('All sessions cleared!') "
نصيحة
في وضع التطوير (DEBUG=true)، Django يولّد SECRET_KEY جديدًا كل تشغيل — مما يسبب مشاكل الجلسات. استخدم مفتاح ثابت في .env.
متوسطة انتهت مدة الجلسة — إعادة توجيه لصفحة الدخول
الحل:

الجلسة تنتهي بعد 7 أيام افتراضيًا. يمكنك تعديل المدة في settings.py:

Python (settings.py) # مدة الجلسة بالثوانٍ (7 أيام = 604800) SESSION_COOKIE_AGE = 86400 * 7

أخطاء الصلاحيات

عالية ليس لديك صلاحية (403 Forbidden)
Permission Denied: You do not have permission to perform this action.
الحل:

النظام يستخدم نظام صلاحيات مخصص عبر access_control. لتفعيل صلاحيات المستخدم:

  1. ادخل على لوحة التحكم: /admin/
  2. اذهب إلى Access Control > User Screen Permissions
  3. أضف صلاحيات للمستخدم المطلوب لكل شاشة ونوع صلاحية (إضافة/تعديل/حذف/عرض)
PowerShell # للتأكد أن المستخدم superuser (جميع الصلاحيات) venv\Scripts\python manage.py shell -c " from django.contrib.auth.models import User u = User.objects.get(username='admin') u.is_superuser = True u.is_staff = True u.save() print('Admin is now superuser!') "

حظر تسجيل الدخول (Throttle)

متوسطة تجاوزت عدد محاولات الدخول المسموح (429)
تجاوزت عدد محاولات الدخول المسموح. يرجى المحاولة بعد بضع دقائق.
الحل:

النظام يحظر IP بعد 10 محاولات فاشلة لمدة 5 دقائق. الانتظار هو الحل الأفضل.

  1. الحل الأول: انتظر 5 دقائق ثم أعد المحاولة
  2. الحل الثاني: احذف الكاش يدويًا (يحتاج إعادة تشغيل السيرفر)
PowerShell # أوقف السيرفر ثم أعد تشغيله — الكاش يُمحى تلقائيًا taskkill /F /IM python.exe venv\Scripts\python run_server.py
ملاحظة
إذا كنت في بيئة متعددة العمليات (Gunicorn مع Redis)، تأكد من استخدام Redis كـ Cache Backend لضمان مشاركة العداد بين العمليات.

أخطاء القوالب (Templates)

عالية خطأ في صيغة القالب (Template SyntaxError)
TemplateSyntaxError: Invalid block tag on line 25: 'endif', expected 'endblock'
الحل:
  1. افتح الملف المعني وتحقق من توازن_brackets ({% %})
  2. تأكد من إغلاق كل {% block %} بـ {% endblock %}
  3. تأكد من إغلاق كل {% if %} بـ {% endif %}
  4. تأكد من إغلاق كل {% for %} بـ {% endfor %}
عالية القالب غير موجود (Template Does Not Exist)
TemplateDoesNotExist: sales/invoice_detail.html
الحل:
  1. تأكد أن القالب موجود في templates/ أو في <app>/templates/
  2. تأكد من تطابق الاسم بالضبط (حساس لحالة الأحرف)
  3. تأكد من أن القالب يرث من قالب أساسي صحيح
متوسطة متغير غير موجود في القالب
TemplateSyntaxError: 'url' requires a non-empty first argument
الحل:

تأكد من تمرير كل المتغيرات المطلوبة من الـ view إلى القالب:

Python (views.py) def my_view(request): context = { 'items': items, 'total': total, } return render(request, 'my_template.html', context)
متوسطة خطأ في custom template tag (can_screen)
ImportError: cannot import name 'can_screen' from 'access_tags'
الحل:
  1. تأكد من تحميل {% load access_tags %} في أعلى القالب
  2. تأكد من أن تطبيق access_control موجود في INSTALLED_APPS
  3. تأكد من وجود ملف access_control/templatetags/access_tags.py

أخطاء الملفات الثابتة (Static Files)

متوسطة الملفات الثابتة (CSS/JS) لا تُحمّل
الأعراض
الصفحة تظهر بدون تنسيق — لا توجد خطوط ألوان ولا JavaScript يعمل.
الحل:
  1. وضع التطوير (DEBUG=True): Django يخدم static تلقائيًا
  2. وضع الإنتاج (DEBUG=False): يجب استخدام Whitenoise أو جمع الملفات
PowerShell # جمع الملفات الثابتة (للإنتاج) venv\Scripts\python manage.py collectstatic --noinput
settings.py # Whitenoise middleware يجب أن يكون بعد SessionMiddleware MIDDLEWARE = [ 'django.middleware.security.SecurityMiddleware', 'whitenoise.middleware.WhiteNoiseMiddleware', # ← الثاني 'django.contrib.sessions.middleware.SessionMiddleware', ... ]

أخطاء النماذج (Forms)

متوسطة خطأ تحقق من النموذج (Form Validation Error)
{'name': ['This field is required.'], 'code': ['This field is required.']}
الحل:
  1. تأكد من ملء جميع الحقول المطلوبة (*)
  2. تأكد من أن بيانات الإرسال صحيحة من ناحية النوع
  3. في AJAX: تحقق من إرسال كل الحقول المطلوبة في الـ JSON
متوسطة خطأ في نوع البيانات (ValueError)
ValueError: invalid literal for int() with base 10: 'abc' ValueError: Could not convert string to Decimal: '1,500.00'
الحل:
  1. تأكد من إدخال أرقام فقط في الحقول الرقمية
  2. لا تستخدم فواصل الأرقام (1,500) — استخدم (1500) أو (1500.00)
  3. في AJAX: تأكد من إرسال الرقم كـ number وليس string

أخطاء الروابط (URLs)

عالية الرابط غير موجود (NoReverseMatch / 404)
NoReverseMatch: Reverse for 'purchase_detail' with arguments '()' not found
الحل:
  1. تأكد من أن الاسم في {% url 'name' %} مطابق لـ name= في urls.py
  2. تأكد من تمرير المعاملات المطلوبة (pk, slug, إلخ)
  3. تحقق من جميع ملفات urls.py للتطبيقات المعنية
Python (urls.py) # تأكد من أن name موجودة path('<int:pk>/', views.detail, name='purchase_detail') # في القالب {% url 'purchase_detail' pk=item.pk %}

أخطاء CSRF

حرج فشل التحقق من CSRF (403 Forbidden)
Forbidden (403): CSRF verification failed. Request aborted.
الحل:
  1. في كل قالب POST: أضف {% csrf_token %} داخل الـ form
  2. في AJAX: اقرأ الكوكيز csrftoken وأرسله في الهيدر
  3. تأكد من أن الكوكيز لم تنته صلاحيتها
HTML <form method="post"> {% csrf_token %} ... </form>
JavaScript (AJAX) // قراءة CSRF token من الكوكيز function getCookie(name) { let v = document.cookie.match('(^|;) ?' + name + '=([^;]*)(;|$)'); return v ? v[2] : null; } fetch(url, { method: 'POST', headers: { 'X-CSRFToken': getCookie('csrftoken'), 'Content-Type': 'application/json', }, body: JSON.stringify(data) });
إذا كنت تستخدم HTTPS عبر وكيل (Proxy)
تأكد من أن CSRF_TRUSTED_ORIGINS يحتوي على العنوان الصحيح. إذا كان الوكيل يحول HTTP إلى HTTPS، أضف https:// للعناوين.

أخطاء الاتصال بالشبكة

حرج Connection Refused — السيرفر لا يستمع
Hmmm… can't reach this page — 127.0.0.1 refused to connect — ERR_CONNECTION_REFUSED
الحل (خطوة بخطوة):
  1. تأكد أن السيرفر يعمل: netstat -ano | findstr :8001
  2. تأكد من البورت: افتح الرابط الصحيح — مثال: http://127.0.0.1:8001
  3. أعد تشغيل السيرفر إذا لم يكن يعمل
  4. تحقق من جدار الحماية إذا كنت تحاول من جهاز آخر
PowerShell # 1. هل السيرفر يعمل؟ netstat -ano | findstr :8001 # يجب أن تظهر: TCP 0.0.0.0:8001 LISTENING # 2. أوقف كل العمليات القديمة وأعد التشغيل taskkill /F /IM python.exe venv\Scripts\python run_server.py
عالية Timeout — السيرفر لا يستجيب
ERR_CONNECTION_TIMED_OUT
الحل:
  1. السيرفر يعمل لكنه بطيء جدًا — راجع قسم الأداء
  2. أو أن البورت محجوب في جدار الحماية
  3. أو أنك تحاول الوصول من خارج الشبكة بدون توجيه منفذ

جدار الحماية (Firewall)

عالية البورت محجوب في جدار الحماية — لا يمكن الوصول من جهاز آخر
الحل — فتح البورت في Windows Firewall:

  2. لتعديل الحد: غيّر RATE_LIMIT في settings.py
  3. لتعطيل: أزل RateLimitMiddleware من MIDDLEWARE

  3. تأكد من تشغيل collectstatic في الإنتاج

هذا يحدث إذا لم يتم تحميل ملف form-validation.js. تأكد من تحميله في القالب:

HTML (base template) <script src="{% static 'js/form-validation.js' %}"></script>

أخطاء FIFO Costing

متوسطة FIFO — رصيد مخزون سلبي أو طبقة غير موجودة
FIFOError: Insufficient stock layers for product XXX
الحل:
  1. هذا يحدث عند محاولة بيع كمية أكبر من المتوفر في طبقات المخزون
  2. تأكد من وجود مخزون كافٍ قبل البيع
  3. استخدم تسوية مخزون لتصحيح الأرصدة
  4. إذا كان الرصيد الفعلي سالباً، استخدم stock_adjustments للتصحيح
متوسطة FIFO — فرق في حساب التكلفة
الحل:
  1. تحقق من أسعار الشراء المدخلة في فواتير المشتريات
  2. FIFO يحتسب التكلفة من أقدم طبقة — تأكد من ترتيب التواريخ
  3. يمكن إعادة حساب التكلفة عبر python manage.py recalculate_fifo
