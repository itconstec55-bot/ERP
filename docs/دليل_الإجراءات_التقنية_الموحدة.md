# القسم الأول: الإعداد الأولي للنظام

## 1.1 إعداد بيانات الشركة

![لوحة التحكم](user_manual/images/02-dashboard.svg)

**المسار:** الإدارة → إعدادات الشركة

**خطوات الإعداد:**

| الخطوة | الشاشة | الإجراء |
|--------|--------|---------|
| 1 | إعدادات الشركة | إدخال: الاسم — العنوان — الرقم الضريبي — الشعار — العملة الأساسية |
| 2 | الفروع | إنشاء فروع الشركة (فرع رئيسي + فروع فرعية) |
| 3 | العملات | تحديد العملة الأساسية (جنيه مصري) + أي عملات أخرى مع أسعار الصرف |

**الحسابات الافتراضية (تُنشأ تلقائياً من Company):**

| الحساب | الكود | الغرض |
|--------|-------|-------|
| حساب العملاء | 1100 | الذمم المدينة |
| حساب الموردين | 3100 | الذمم الدائنة |
| حساب الخزينة | 1500 | النقدية |
| حساب المبيعات | 4100 | إيرادات المبيعات |
| حساب المشتريات | 5100 | تكلفة المشتريات |
| حساب المخزون | 1300 | المخزون |
| حساب VAT | 2210 | ضريبة القيمة المضافة |
| حساب الخصم والتحصيل | 2220 | ضريبة الخصم عند المصدر |

## 1.2 إعداد شجرة الحسابات

![شجرة الحسابات](user_manual/images/10-chart-accounts.svg)

**المسار:** المعالجة المالية → الحسابات → دليل الحسابات

**خطوات الإعداد:**

| الخطوة | الإجراء |
|--------|---------|
| 1 | استيراد شجرة الحسابات من ملف Excel (أو الإنشاء اليدوي) |
| 2 | التأكد من وجود 5 أنواع رئيسية: أصول — خصوم — حقوق ملكية — إيرادات — مصروفات |
| 3 | ربط الحسابات البنكية بحسابات الدليل |
| 4 | ربط الحسابات الضريبية (VAT + الخصم والتحصيل) |

**الشجرة الهرمية:**

```
1 — الأصول
├── 11 — الأصول المتداولة
│   ├── 1100 — حساب العملاء (ذمم مدينة)
│   ├── 1200 — المخزون
│   └── 1300 — النقدية في الصندوق
├── 15 — الأصول الثابتة
│   ├── 1500 — الخزينة / البنك
│   └── 1600 — أصول ثابتة (آلات، سيارات)
2 — الخصوم
├── 21 — الخصوم المتداولة
│   ├── 2100 — حساب الموردين (ذمم دائنة)
│   └── 2210 — ضريبة القيمة المضافة
├── 2220 — ضريبة الخصم والتحصيل
3 — حقوق الملكية
├── 3100 — رأس المال
├── 3200 — أرباح محتجزة
4 — الإيرادات
├── 4100 — إيرادات المبيعات
5 — المصروفات
├── 5100 — تكلفة البضاعة المباعة
├── 5200 — مصروفات تشغيلية
```

## 1.4 إعداد المستخدمين والصلاحيات

**المسار:** الإدارة → المستخدمون والصلاحيات

**الأدوار الافتراضية (5 أدوار):**

| الدور | الصلاحيات |
|-------|----------|
| **مدير النظام** | كل الشاشات — كل الإجراءات (عرض/إضافة/تعديل/حذف/طباعة/تصدير) |
| **محاسب** | الحسابات + الخزينة + سندات القبض/صرف + الأصول + التقارير |
| **أمين مخزن** | المخازن + المنتجات + طلبات الشراء + إذون الاستلام + المرتجعات |
| **مبيعات** | العملاء + فواتير المبيعات + أوامر البيع + المرتجعات + الفواتير الضريبية |
| **مشاهدة فقط** | كل الشاشات — عرض + طباعة + تصدير فقط |

**خطوات الإعداد:**

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء حساب المستخدم (اسم مستخدم + كلمة مرور) |
| 2 | تعيين الدور للمستخدم |
| 3 | تحديد الفروع المخوّلة (إذا كان المستخدم يعمل في فرع محدد) |
| 4 | تحديد المخازن المخوّلة (لأمين المخزن) |
| 5 | تحديد أنواع الحسابات المسموح بها (اختياري) |

> **مهم:** نظام الصلاحيات يعتمد على `access_control` فقط — لا يعتمد على مجموعات Django

# القسم الثاني: إنتاج الخرسانة

> **الدورة الأساسية** — كل شيء يبدأ من هنا

## 2.1 خلطات الخرسانة (Mix Designs)

**المسار:** إنتاج الخرسانة → خلطات الخرسانة

| البند | التفاصيل |
|-------|---------|
| **الغرض** | تعريف تركيبة الخرسانة (نسبة الأسمنت + رمل + حصى + ماء + مواد مضافة) |
| **ال権利** | `concrete_production.concretemixdesign` |

**خطوات الإنشاء:**

| الخطوة | الحقل | التفاصيل |
|--------|-------|---------|
| 1 | اسم الخلطة | مثال: C25 — خلطة 25 ميجاباسكال |
| 2 | الصنف | ربطه بصنف في المشتريات (منتج خرسانة جاهز) |
| 3 | مكونات الخلطة | كل مكون: المادة + الكمية لكل متر مكعب + التكلفة التقديرية |
| 4 | قوة الضغط المطلوبة | بالنيوتن/مم2 |

**القيود التلقائية:**
- كل مكون لازم يكون صنف مسجل في المشتريات
- التكلفة الإجمالية تُحسب تلقائياً = مجموع (الكمية × سعر الوحدة)

---

## 2.2 طلبات العملاء (Customer Requests)

**المسار:** إنتاج الخرسانة → طلبات العملاء

| البند | التفاصيل |
|-------|---------|
| **الغرض** | تسجيل طلب العميل (نوع الخلطة + الكمية + مكان التسليم) |
| **ال権利** | `concrete_production.customerrequest` |

**خطوات الإنشاء:**

| الخطوة | الحقل | التفاصيل |
|--------|-------|---------|
| 1 | العميل | اختيار من قائمة العملاء (مسجل في المبيعات) |
| 2 | طلب العميل | نص الطلب |
| 3 | خلطة الخرسانة | اختيار من الخلطات المعرّفة |
| 4 | الكمية المطلوبة | بالمتر المكعب |
| 5 | موقع التسليم | العنوان |
| 6 | التاريخ المطلوب | تاريخ التسليم المتوقع |

**الحالة:** `مسودة` → `معتمد` → `تم التحويل لأمر إنتاج`

---

## 2.3 أوامر الإنتاج (Production Orders)

**المسار:** إنتاج الخرسانة → أوامر الإنتاج

| البند | التفاصيل |
|-------|---------|
| **الغرض** | أمر إنتاج رسمي يُحوّل طلب العميل إلى أوامر عمل |
| **ال権يل** | `concrete_production.productionorder` |

**خطوات الإنشاء:**

| الخطوة | الحقل | التفاصيل |
|--------|-------|---------|
| 1 | طلب العميل | ربط بأمر العميل (اختياري — قد يكون مباشرة) |
| 2 | فاتورة المبيعات | ربط بفاتورة مبيعات (إن وُجدت) |
| 3 | الفرع | تحديد فرع الإنتاج |
| 4 | خلطة الخرسانة | اختيار الخلطة |
| 5 | الكمية المطلوبة | بالمتر المكعب |
| 6 | سعر المتر المكعب | سعر البيع |

**القيود التلقائية عند الترحيل:**
- يُنشأ قيد محاسبي: مدين حساب العميل — دائن إيرادات المبيعات
- يُنشأ بند فاتورة مبيعات تلقائياً (إذا كانت مربوطة)

---

## 2.4 المتابعة اليومية والدفعات الإنتاجية

**المسار:** إنتاج الخرسانة → المتابعة اليومية / الدفعات الإنتاجية

| الشاشة | الوظيفة |
|--------|---------|
| المتابعة اليومية | مراقبة الإنتاج اليومي — كم متر تم إنتاجه فعلياً |
| الدفعات الإنتاجية | تسجيل كل دفعة إنتاج (الكمية الفعلية + расход الأسمنت الفعلي) |
| الشاحنات | إدارة أسطول الشاحنات (المقاسات + الحالة) |
| جداول التسليم | جدولة التوقيتات لتسليم الخرسانة للمواقع |

---

## 2.5 تكاليف الإنتاج

**المسار:** إنتاج الخرسانة → تكلفة المتر المكعب

| البند | التفاصيل |
|-------|---------|
| **الغرض** | حساب التكلفة الفعلية لكل متر مكعب من الخرسانة |
| **المعادلة** | التكلفة = (تكلفة الأسمنت + تكلفة الرمل + تكلفة الحصى + تكلفة المواد المضافة + تكلفة النقل) ÷ الكمية |

**التكاليف المحسوبة تلقائياً:**
- تكلفة الأسمنت = (الكمية المستهلكة × سعر الوحدة)
- تكلفة المعدات = (ساعات التشغيل × تكلفة الساعة)
- تكلفة العمالة = (عدد العمال × الأجر اليومي)

---

## 2.6 السيلو والجرد اليومي

**المسار:** إنتاج الخرسانة → سيلو / جرد الأسمنت اليومي

| الشاشة | الوظيفة |
|--------|---------|
| السيلو | إدارة خزائن الأسمنت (السعة — الرصيد الحالي — المورد) |
| جرد الأسمنت اليومي | تسجيل الإيداعات والسحوبات اليومية للأسمنت |

# القسم الثالث: المشتريات

> **من طلب الشراء إلى فاتورة المشتريات**

## 3.1 دورة الشراء الكاملة

```
طلب شراء → عرض أسعار → أمر شراء → إذن استلام → فاتورة مشتريات
```

![فوترة المشتريات](user_manual/images/05-invoice.svg)

## 3.2 طلبات الشراء (Requisitions)

**المسار:** المشتريات → طلبات الشراء

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء طلب شراء: الصنف + الكمية + مركز التكلفة + السبب |
| 2 | المراجعة والاعتماد من المدير المخوّل |
| 3 | التحويل لعرض أسعار أو أمر شراء مباشر |

**الحالة:** `مسودة` → `معتمد` → `تم التحويل`

## 3.3 طلبات عروض الأسعار (RFQ)

**المسار:** المشتريات → عروض الأسعار

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء طلب عرض سعر من مورد أو عدة موردين |
| 2 | إدخال أسعار الموردين لكل صنف |
| 3 | مقارنة الأسعار واختيار الأفضل |
| 4 | التحويل لأمر شراء |

## 3.4 أوامر الشراء (Purchase Orders)

**المسار:** المشتريات → أوامر الشراء

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء أمر شراء: المورد + الأصناف + الأسعار |
| 2 | طباعة أمر الشراء وإرساله للمورد |
| 3 | متابعة التسليم |

## 3.5 إذون الاستلام (Goods Received Notes)

**المسار:** المشتريات → إذون الاستلام

| الخطوة | الإجراء |
|--------|---------|
| 1 | استلام البضاعة من المورد |
| 2 | التحقق من الكميات مقابل أمر الشراء |
| 3 | إنشاء إذن الاستلام (يزيد المخزون تلقائياً) |
| 4 | ربطه بأمر الشراء |

**القيود التلقائية:**
- حركة مخزون واردة (receive) لكل صنف
- تحديث أرصدة المنتجات في المخزن

## 3.6 فواتير المشتريات (Purchase Invoices)

**المسار:** المشتريات → فواتير المشتريات

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء الفاتورة: المورد + الأصناف + الأسعار |
| 2 | حفظ كمسودة |
| 3 | اعتماد الفاتورة |
| 4 | ترحيل الفاتورة (إنشاء القيد المحاسبي) |

**القيد المحاسبي عند الترحيل:**

| الحساب | مدين | دائن |
|--------|------|------|
| تكلفة المشتريات (5100) | المبلغ الإجمالي | — |
| ض. القيمة المضافة (2210) | مبلغ VAT | — |
| حساب الموردين (3100) | — | المبلغ الإجمالي + VAT |

## 3.7 مرتجعات المشتريات

**المسار:** المشتريات → مرتجعات المشتريات

**القيود التلقائية:**
- عكس القيد المحاسبي لفاتورة المشتريات
- حركة مخزون خارجة (return)

# القسم الرابع: المبيعات

> **من العميل إلى فاتورة المبيعات**

## 4.1 إعداد العملاء

**المسار:** العمليات اليومية → المبيعات → العملاء

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء ملف العميل: الاسم — النوع — الرقم الضريبي — العنوان — الاتصال |
| 2 | تحديد حد الائتمان (الحد الأقصى للمديونية المسموحة) |
| 3 | ربطه بحساب محاسبي في الدليل |

## 4.2 دورة المبيعات الكاملة

```
طلب العميل → فاتورة مبيعات → ترحيل → تحصيل المبلغ (سند قبض)
```

![فوترة المبيعات](user_manual/images/05-invoice.svg)

## 4.3 فواتير المبيعات (Sales Invoices)

**المسار:** العمليات اليومية → المبيعات → فواتير المبيعات

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء الفاتورة: العميل + الأصناف + الكميات + الأسعار |
| 2 | تحديد: هل فاتورة ضريبية (VAT 14%)؟ |
| 3 | تحديد: هل ض. خصم وتحصيل (1% / 3% / 5%)؟ |
| 4 | حفظ كمسودة |
| 5 | اعتماد الفاتورة |
| 6 | ترحيل الفاتورة (إنشاء القيد المحاسبي + حركة المخزون) |

**القيد المحاسبي عند الترحيل:**

| الحساب | مدين | دائن |
|--------|------|------|
| حساب العميل (1100) | المبلغ الإجمالي + VAT | — |
| إيرادات المبيعات (4100) | — | المبلغ قبل الضريبة |
| ض. القيمة المضافة (2210) | — | مبلغ VAT |
| تكلفة البضاعة المباعة (5100) | تكلفة الأصناف | — |
| المخزون (1300) | — | تكلفة الأصناف |

> **مهم:** الترحيل ينشئ حركة خروج من المخزون تلقائياً لكل صنف في الفاتورة

**المراحل الإلزامية:**

```
💾 حفظ (مسودة) → ✔ اعتماد → ⇪ ترحيل → 🖨 طباعة
```

## 4.4 مرتجعات المبيعات

**المسار:** العمليات اليومية → المبيعات → مرتجعات المبيعات

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء مرتجع: ربط بالفاتورة الأصلية + أسباب المرتجع |
| 2 | تحديد الكميات المرتجعة |
| 3 | ترحيل المرتجع |

**القيود التلقائية:**
- عكس قيد فاتورة المبيعات (مدين الإيرادات — دائن العميل)
- حركة مخزون واردة (استرداد للمخزن)

## 4.5 أوامر البيع (Sales Orders)

**المسار:** العمليات اليومية → المبيعات → أوامر البيع

| الخطوة | الإجراء |
|--------|---------|
| 1 | إنشاء أمر البيع: العميل + الأصناف + الأسعار |
| 2 | اعتماد أمر البيع |
| 3 | التحويل لفاتورة مبيعات عند التسليم |

## 4.6 إشعارات الدائن / المدين (Credit Notes)

**المسار:** العمليات اليومية → المبيعات → إشعارات الدائن / المشتريات → إشعارات المدين

| النوع | الاستخدام |
|-------|----------|
| إشعار دائن | تخفيض مديونية العميل (خصم / تعديل) |
| إشعار مدين | زيادة مديونية المورد (خصم / تعديل) |

# القسم الخامس: الدورة النقدي

> **سندات القبض والصرف — من الفاتورة إلى التحصيل**

## 5.1 سند القبض (Payment Receipt — Inflow)

**المسار:** العمليات اليومية → الخزينة → سندات القبض والصرف

**خطوات الإنشاء:**

| الخطوة | الحقل | التفاصيل |
|--------|-------|---------|
| 1 | رقم السند | يُولَّد تلقائياً PR-XXXXX |
| 2 | التاريخ | تاريخ اليوم |
| 3 | المبلغ | المبلغ المقبوض |
| 4 | طريقة الدفع | نقداً / تحويل بنكي / شيك / فيزا |
| 5 | العميل | اختيار العميل (يُحمّل فواتيره المستحقة) |
| 6 | فاتورة المبيعات | اختيار الفاتورة (يظهر المتبقي تلقائياً) |
| 7 | البنك / الخزينة | تحديد أين سيذهب المبلغ |
| 8 | البيان | وصف العملية |

**القيد المحاسبي عند الترحيل:**

| الحساب | مدين | دائن |
|--------|------|------|
| الخزينة / البنك (1500) | المبلغ | — |
| حساب العميل (1100) | — | المبلغ |

**التحديثات التلقائية:**
- `SalesInvoice.paid_amount` += المبلغ
- `Customer.current_balance` -= المبلغ

## 5.2 سند الصرف (Payment Receipt — Outflow)

**خطوات الإنشاء:** (مطابق لسند القبض مع فروقات)

|--------|---------|

**القيد المحاسبي:**

| الحساب | مدين | دائن |
|--------|------|------|
| حساب الموردين (3100) | المبلغ | — |
| الخزينة / البنك (1500) | — | المبلغ |

# القسم الثامن: المعالجة المالية

## 8.1 القيود اليومية اليدوية

![القيود اليومية](user_manual/images/04_journal_list.svg)

**المسار:** المعالجة المالية → الحسابات → القيود اليومية

**خطوات الإنشاء:**

| الخطوة | الحقل | التفاصيل |
|--------|-------|---------|
| 1 | نوع القيد | عام / شراء / بيع / قبض / صرف / إهلاك / رواتب |
| 2 | التاريخ | تاريخ القيد |
| 3 | الوصف | وصف محتوى القيد |
| 4 | البنود | كل بند: حساب + مدين أو دائن |

**القاعدة الذهبية:** مجموع المدين = مجموع الدائن (لا يُقبل قيد غير مُتوازن)

**القيود التلقائية (يولّدها النظام):**

| المستند | القيد |
|---------|-------|
| فاتورة مبيعات | مدين العميل — دائن الإيرادات + VAT |
| فاتورة مشتريات | مدين المشتريات + VAT — دائن المورد |
| سند قبض | مدين الخزينة — دائن العميل |
| سند صرف | مدين المورد — دائن الخزينة |
| إهلاك أصل | مدين مصروف الإهلاك — دائن إهلاك متراكم |
| رواتب | مدين مصروف الرواتب — دائن الخزينة |

## 8.2 القيود الدورية (Recurring Entries)

**المسار:** المعالجة المالية → الحسابات → القيود الدورية

| الاستخدام | المثال |
|----------|--------|
| إيجار شهري | قيد تكراري كل شهر |

## 8.3 ميزان المراجعة (Trial Balance)

![ميزان المراجعة](user_manual/images/07-report.svg)

**المسار:** المعالجة المالية → الحسابات → ميزان المراجعة

| البند | التفاصيل |
|-------|---------|
| **الغرض** | التحقق من توازن القيود (مجموع المدين = مجموع الدائن) |
| **المعايير** | من تاريخ — إلى تاريخ — نوع الحساب |
| **العرض** | الكود + اسم الحساب + مدين + دائن + الرصيد |

## 8.4 إغلاق السنة المالية

**المسار:** المعالجة المالية → إغلاق السنة المالية

# القسم العاشر: الإدارة والنظام

|---------|---------|

| التقرير | المحتوى |
|---------|---------|

# ملخص القواعد الذهبية

| # | القاعدة |
|---|---------|
| 1 | **كل فاتورة تنشئ قيداً محاسبياً تلقائياً** عند الترحيل |
| 2 | **القيد يجب أن يكون مُتوازناً** (مجموع المدين = مجموع الدائن) |
| 3 | **الترحيل يُقفل المستند** — لا يمكن التعديل المباشر بعده |
| 4 | **لتصحيح مستند مُرحّل** — أنشئ قيداً عكسياً أو مرتجعاً |
| 5 | **الإلغاء متاح للمسودة فقط** قبل الترحيل |
| 6 | **كل انتقال يُسجَّل** في سجل الإجراءات (من + متى) |
| 7 | **سند القبض/الصرف يُحدّث** رصيد العميل/المورد تلقائياً |
| 8 | **إذن الاستلام يزيد المخزون** — فاتورة المبيعات تنقصه |
| 9 | **لا يمكن حذف سجل له قيود محاسبية** — الحماية تمنع الحذف |
| 10 | **كل مستند يمر بـ: مسودة → اعتماد → ترحيل** |
