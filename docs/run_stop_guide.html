متطلبات التشغيل

💡
ملاحظة هامة

يجب التأكد من تثبيت جميع المتطلبات التالية قبل محاولة تشغيل النظام. يُنصح باستخدام أحدث إصدار متاح لكل برنامج.

🎰

Python 3.12

بيئة تشغيل لغة بايثون. يُفضّل استخدام أحدث إصدار من Python 3.12 مع pip المحدّث.

🗃

PostgreSQL 15+

قاعدة البيانات للإنتاج. يمكن استخدام SQLite للتطوير والاختبار.

🚀

Redis 7+

مطلوب لتشغيل المهام المجدولة وتخزين مؤقت للكش.

💻

ذاكرة RAM

4 جيجابايت كحد أدنى. 8 جيجابايت موصى به للإنتاج.

📁

مساحة التخزين

500 ميجابايت للتطبيق + حجم قاعدة البيانات.

📊

Docker (اختياري)

الإصدار 20.10+ و Docker Compose v2 للنشر عبر الحاويات.

طرق التشغيل

يدعم النظام ثلاث طرق مختلفة للتشغيل. اختر الطريقة المناسبة حسب بيئتك ومتطلباتك.

💻 الطريقة الأولى: تشغيل التطوير (Development)

هذه الطريقة مناسبة للتطوير والاختبار المحلي. تستخدم سيرفر Django المدمج مع ميزة إعادة التحميل التلقائي.

فتح Terminal

افتح سطر الأوامر (PowerShell أو CMD) وانتقل إلى مجلد المشروع.

تفعيل البيئة الافتراضية

فعّل بيئة Python الافتراضية لتثبيت التبعيات المعزولة.

تعيين متغيرات البيئة

قم بتعيين المتغيرات اللازمة أو استخدم ملف .env المُعد مسبقاً.

تشغيل الترحيلات

طبّق التغييرات على بنية قاعدة البيانات.

تشغيل السيرفر

شغّل سيرفر التطوير على العنوان 0.0.0.0:8012.

فتح المتصفح

افتح المتصفح وانتقل إلى الرابط المحدد للوصول إلى النظام.

Terminal - PowerShell
# الانتقال إلى مجلد المشروع cd D:\accounting_system # تفعيل البيئة الافتراضية venv\Scripts\activate # تعيين متغيرات البيئة $env:DJANGO_SETTINGS_MODULE="config.settings.development" # تشغيل الترحيلات python manage.py migrate # تشغيل سيرفر التطوير python manage.py runserver 0.0.0.0:8012
🔗
رابط الدخول

بعد تشغيل السيرفر بنجاح، افتح المتصفح على: http://localhost:8012

📋 الطريقة الثانية: تشغيل الإنتاج عبر Docker

طريقة النشر عبر الحاويات. الأفضل للإنتاج والتكامل المستمر. توفر بيئة موحّدة وقابلة للتكرار.

⚠
ملاحظة

يجب تثبيت Docker و Docker Compose على جهازك قبل المتابعة. تأكد من تشغيل Docker daemon.

التأكد من تثبيت Docker و Docker Compose

تحقق من الإصدار المثبّت عبر الأوامر التالية.

إعداد ملف البيئة

انسخ ملف .env.example إلى .env وعدّل الإعدادات حسب بيئتك.

بناء وتشغيل الحاويات

شغّل أمر البناء والتشغيل. سيقوم Docker Compose ببناء الصور وتشغيل جميع الخدمات.

التحقق من التشغيل

افتح المتصفح على المنفذ المحدد للتأكد من عمل النظام بشكل صحيح.

Terminal - Docker
# التحقق من تثبيت Docker docker --version docker compose version # نسخ ملف البيئة copy .env.example .env # تعديل ملف الإعدادات # افتح ملف .env وعدّل القيم حسب بيئتك # بناء وتشغيل الحاويات docker compose up -d --build # عرض حالة الحاويات docker compose ps # عرض السجلات docker compose logs -f
📷

نافذة Terminal تعرض حالات الحاويات النشطة

لقطة شاشة: إعدادات .env

⚡ الطريقة الثالثة: تشغيل الإنتاج عبر Waitress (Windows)

مناسب للنشر على خوادم Windows في بيئة الإنتاج. يستخدم Waitress كبديل احترافي لسيرفر التطوير.

التأكد من تثبيت التبعيات

ثبّت تبعيات الإنتاج من ملف requirements.txt.

إعداد ملف البيئة

تأكد من وجود ملف .env بالإعدادات الصحيحة لبيئة الإنتاج.

تشغيل Waitress

شغّل سيرفر Waitress عبر ملف التشغيل المُعد.

Terminal - Waitress Production
# تفعيل البيئة الافتراضية venv\Scripts\activate # تثبيت التبعيات pip install -r requirements.txt # تشغيل Waitress python run_waitress.py
🚨
تحذير أمني

لا تستخدم سيرفر التطوير (runserver) في بيئة الإنتاج. استخدم Waitress أو Docker مع Nginx دائماً.

3

الإيقاف الآمن

الإيقاف السليم يضمن عدم فقدان البيانات وسلامة قاعدة البيانات.

🚨
مهم جداً

لا تُغلق النافذة مباشرة بدون إيقاف السيرفر بشكل صحيح. استخدم دائماً الخطوات التالية لضمان سلامة البيانات.

📥 الإيقاف العادي

الضغط على Ctrl + C

في نافذة Terminal التي يعمل فيها السيرفر، اضغط Ctrl + C لإيقاف السيرفر بشكل سلس.

انتظار اكتمال الإيقاف

انتظر حتى يظهر التأكيد على إيقاف جميع العمليات قبل إغلاق النافذة.

📋 الإيقاف عبر Docker
Terminal - Docker
# إيقاف جميع الحاويات docker compose down # إيقاف وحذف البيانات المؤقتة docker compose down -v
💾 عمل نسخة احتياطية قبل الإيقاف
💡
توصية

يُنصح دائماً بعمل نسخة احتياطية قبل أي إجراء تعديل أو إيقاف طويل المدة.

Terminal - Backup
# عمل نسخة احتياطية من قاعدة البيانات python manage.py backup # النسخة الاحتياطية ستُحفظ في مجلد: # backups/

فحص صحة النظام

يوفر النظام نقاط نهاية مخصصة لمراقبة حالته والتأكد من عمل جميع المكونات بشكل سليم.

النقطة الرابط الوصف الحالة
فحص الصحة العام /health/ فحص شامل لجميع مكونات النظام متاح
لوحة المراقبة /monitoring/ لوحة مراقبة تفاعلية مع رسوم بيانية متاح
فحص قاعدة البيانات /health/db/ التحقق من اتصال قاعدة البيانات متاح
فحص Redis /health/redis/ التحقق من خدمة Redis اختياري
أوامر فحص الصحة
# فحص الصحة عبر المتصفح http://localhost:8012/health/ # فحص الصحة عبر سطر الأوامر (curl) curl -s http://localhost:8012/health/ # فتح لوحة المراقبة http://localhost:8012/monitoring/

استكشاف الأخطاء وإصلاحها

دليل سريع لحل أكثر الأخطاء شيوعاً التي قد تواجهك أثناء تشغيل النظام.

🔴 المشكلة 1: السيرفر لا يعمل

في حال عدم تشغيل السيرفر أو ظهور رسالة خطأ عند المحاولة:

فحص السجلات

راجع ملف السجلات للتحقق من سبب الخطأ.

إعادة تشغيل السيرفر

أوقف وأعد تشغيل السيرفر.

التحقق من المنفذ

تأكد من عدم استخدام المنفذ 8012 من برنامج آخر.

الخطوات العلاجية
# مراجعة السجلات type logs\accounting.log # التحقق من المنفذ المستخدم netstat -ano | findstr 8012 # إعادة تشغيل السيرفر python manage.py runserver 0.0.0.0:8012
🔴 المشكلة 2: خطأ في اتصال قاعدة البيانات

رسالة خطأ تشير إلى عدم القدرة على الاتصال بقاعدة البيانات:

التحقق من تشغيل PostgreSQL

تأكد من أن خدمة PostgreSQL تعمل بشكل صحيح.

التحقق من الإعدادات

راجع إعدادات الاتصال في ملف .env أو settings.py.

اختبار الاتصال

استخدم أداة مثل pgAdmin لاختبار الاتصال يدوياً.

الخطوات العلاجية
# التحقق من حالة PostgreSQL pg_isready -h localhost -p 5432 # اختبار الاتصال بقاعدة البيانات psql -U postgres -d accounting_db # إعادة ترحيل قاعدة البيانات python manage.py migrate
🟠 المشكلة 3: مشاكل الملفات والصلاحيات

أخطاء تتعلق بقراءة أو كتابة الملفات:

الخطوات العلاجية
# جمع الملفات الساكنة python manage.py collectstatic --noinput # إنشاء مجلد السجلات إذا لم يكن موجوداً if not exist logs mkdir logs # التحقق من صلاحيات الكتابة icacls D:\accounting_system
🟠 المشكلة 4: المهام المجدولة لا تعمل

في حال عدم عمل المهام الآلية أو التقارير المجدولة:

الخطوات العلاجية
# التحقق من حالة Redis redis-cli ping # مراجعة سجلات Celery (إذا كان مستخدماً) type logs\celery.log # إعادة تشغيل خدمة المهام المجدولة celery -A config worker --loglevel=info
📖 مرجع سريع للأخطاء الشائعة
الخطأ السبب المحتمل الحل
ModuleNotFoundError التبعيات غير مثبّتة pip install -r requirements.txt
OperationalError: connection قاعدة البيانات غير متاحة تحقق من PostgreSQL
Permission denied صلاحيات المجلدات تحقق من صلاحيات الكتابة
Address already in use المنفذ مستخدم مسبقاً غيّر المنفذ أو أوقف البرنامج الآخر
Static files not found الملفات الساكنة غير مجمّعة python manage.py collectstatic
