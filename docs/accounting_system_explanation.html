1 مقدمة عن النظام

نظام ERP محاسبي متكامل مصمم خصيصاً لشركات إنتاج الخرسانة الجاهزة في مصر. يوفر النظام إدارة شاملة للمبيعات، المشتريات، الإنتاج، المحاسبة، والموارد البشرية في منصة واحدة.

2 الفلسفة المحاسبية

نظام القيد المزدوج (Double Entry)

يعتمد النظام على مبدأ القيد المزدوج حيث كل معاملة تؤثر على حسابين على الأقل بقيمتين متساويتين. هذا يضمن دقة البيانات واتساق القوائم المالية.

# مثال: شراء مواد خام بمبلغ 100,000 ج.م
# القيد المزدوج:
#   مدين:  حساب مواد خام (أصول)          100,000
#   دائن:  حساب مورد (خصوم)              100,000

أساس الاستحقاق (Accrual Basis)

تُسجّل الإيرادات عند استحقاقها والمصروفات عند نشوئها بغض النظر عن توقيت استلام أو دفع النقد.

المحورالنظام النقدينظام الاستحقاق
تسجيل الإيرادعند استلام النقدعند إصدار الفاتورة
تسجيل المصروفعند دفع النقدعند استلام الفاتورة
الحسابات المستحقةلا تُستخدممدين/دائن مستحق
الدقة الماليةأقلأعلى

التوافق مع المعايير المحاسبية المصرية

  • التوافق مع معايير الإبلاغ المالي الدولية (IFRS) المُعتمد في مصر
  • التوافق مع معايير المحاسبة المصرية (Egyptian GAAP)
  • متابعة التحديثات الصادرة من هيئة سوق المال المصرية
  • التوافق مع أنظمة هيئة الزكاة والضريبة والجمارك

التوافق مع قوانين الضرائب المصرية

  • ضريبة القيمة المضافة (VAT) بنسبة 14%
  • ضريبة الدخل (ال_profit tax) بنسبة 22.5%
  • ضريبة الدمغة على المستندات والعقود
  • النظام الفرعي للفواتير الإلكترونية (E-Invoicing)
✓ ملاحظة

النظام مصمم ليكون متوافقاً مع جميع المتطلبات التنظيمية للمصالحة الضريبية المصرية (ETA) عند الإصدار 2026.

3 الدورة المحاسبية كاملة

3.1 دورة المبيعات

تبدأ من استلام طلب العميل وانتهاءً بالقبض والمconciliation.

عميل ← عرض سعر ← أمر بيع ← أمر إنتاج
فاتورة بيع ← مقبوضات ← قيد محاسبي ← تقرير

المراحل التفصيلية:

المقطعالمستندالحساب المتأثرالتلقائي
1عرض سعرلا قيد — معلوماتية فقطمعلومة
2أمر بيعحساب مبيعات مستحقة + إيراد مبيعاتتلقائي
3أمر إنتاجتحويل أوامر الإنتاجتلقائي
4فاتورة بيعمدين: حساب عميل / دائن: إيراد + ضريبةتلقائي
5سند قبضمدين: صندوق/بنك / دائن: حساب عميلتلقائي

3.2 دورة المشتريات

تبدأ من تحديد الاحتياج وانتهاءً بالدفع.

مورد ← طلب شراء ← أمر شراء ← استلام بضاعة
فاتورة مشتريات ← مدفوعات ← قيد محاسبي ← تقرير

المراحل التفصيلية:

المقطعالمستندالحساب المتأثرالتلقائي
1طلب شراءلا قيد — موافقة إداريةمعلومة
2أمر شراءحساب مشتريات مستحقة + التزامتلقائي
3سند استلام بضاعةمدين: مخزون / دائن: مشتريات مستحقةتلقائي
4فاتورة مشترياتمدين: مورد / دائن: بنك/صندوقتلقائي
5سند صرفمدين: مورد / دائن: صندوق/بنكتلقائي

3.3 دورة إنتاج الخرسانة

الدورة الفريدة لنظام إنتاج الخرسانة الجاهزة.

طلب عميل ← تصميم الخلطة (Mix Design) ← أمر إنتاج
خلط batching ← شحن mixers ← عيانة QA/QC ← فاتورة

مراحل الإنتاج التفصيلية:

  1. استلام طلب العميل: تحديد نوع الخرسانة، الكمية، موعد التسليم، وموقع التسليم
  2. تصميم الخلطة (Mix Design): تحديد نسب المواد الخام (أسمنت، رمل، حصى، ماء، إضافات) بناءً على المعامل المطلوب (fcu)
  3. أمر الإنتاج: إصدار أمر إنتاج بناءً على طلب العميل وتصميم الخلطة
  4. مرحلة الخلط (Batching): وزن المواد الخام وخلطها في المحطة (Batching Plant) وفقاً لتصميم الخلطة
  5. 阶段 الشحن (Mixers): نقل الخلطة في خلايا اله Cyclone وتحديد وقت الشحن والتسليم المتوقع
  6. العيانة والجودة (QA/QC): أخذ عينات واختبار الانضغاط بعد 7 و 28 يوماً لضمان الجودة
  7. إصدار الفاتورة: فاتورة بيع بناءً على الكمية المُسلّمة فعلياً والحد الأدنى للتحميل
ملاحظة تقنية

يحتوي النظام على نموذج حساب التكلفة لكل متر مكعب من الخرسانة بناءً على تكاليف المواد الخام والطاقة العمالة المباشرة والمصروفات غير المباشرة.

3.4 الدورة المحاسبية

قيود يومية ← ترحيل ← ميزان مراجعة
قيود تسوية ← ميزان مراجعة نهائي ← قائمة دخل ← ميزانية

مراحل الدورة المحاسبية الشهرية:

المقطعالعمليةالوصف
1تسجيل القيود اليوميةتسجيل جميع المعاملات في دفتر اليومية العام
2الترحيلنقل القيود من دفتر اليومية إلى دفاتر الأستاذ
3ميزان المراجعةإعداد جدول يوضح أرصدة جميع الحسابات للتحقق من التوازن
4القيود التصحيحية والتسويةتسجيل قيود الإهلاك، الاستهلاكات، الإيرادات المستحقة، المصروفات المستحقة
5ميزان المراجعة النهائيإعادة إعداد الميزان بعد القيود التصحيحية
6إعداد القوائم الماليةقائمة الدخل، الميزانية العمومية، قائمة التدفقات النقدية
7الإقفالإقفال الحسابات المؤقتة ونقل الأرباح إلى حقوق الملكية

6 الضرائب في النظام

6.1 ضريبة القيمة المضافة (VAT) - 14%

يلتزم النظام بحساب ضريبة القيمة المضافة بنسبة 14% على جميع المعاملات الخاضعة للضريبة.

حساب VAT على فاتورة بيع:

# مثال: فاتورة بيع خرسانة
القيمة الإجمالية (شاملة الضريبة) = 570,000 ج.م
نسبة الضريبة = 14%

# الحساب:
القيمة قبل الضريبة = 570,000 / 1.14 = 500,000 ج.م
مبلغ الضريبة = 500,000 × 0.14 = 70,000 ج.م

# القيود المحاسبية:
مدين: الذمم المدينة                     570,000
دائن: إيراد مبيعات الخرسانة              500,000
دائن: ضريبة القيمة المضافة المستحقة      70,000

6.2 ضريبة الخصم (Withholding Tax)

يقوم النظام باحتساب ضريبة الخصم عند الدفع للموردين وفقاً للنسب المحددة قانونياً.

النوعالنسبةالحد الأدنى
خدمات بناءً على العقود1%لا يوجد
إيجار المباني10%لا يوجد
إيجار الأراضي5%لا يوجد
الفنية والاستشارية20%لا يوجد
الدعاية والإعلان5%لا يوجد
⚠ ملاحظة

يقوم النظام تلقائياً بإنشاء سند الصرف مع احتساب ضريبة الخصم المناسبة وتسجيل القيد المحاسبي للمبلغ الصافي وقيمة الضريبة المحتسبة.

6.3 الفاتورة الإلكترونية (E-Invoicing)

يدعم النظام التكامل مع منصة الفواتير الإلكترونية لهيئة الزكاة والضريبة والجمارك (ETA).

  • إنشاء فاتورة بصيغة JSON/XML متوافقة مع متطلبات ETA
  • إرسال الفاتورة مباشرة عبر API الخاص بـ ETA
  • حفظ UUID (المعرف الفريد) لكل فاتورة
  • تتبع حالة الفاتورة (مقبولة/مرفوضة)
  • دعم الطرح الرئيسي (Main Sale Invoice)
  • دعم الفواتير التعديلية (Credit/Debit Notes)

  • صلاحيات قسم: كل مستخدم يرى فقط سجلات قسمه (مبيعات، مشتريات، إنتاج...)
  • صلاحيات منطقة: المندوبون يرون فقط عملاء منطقتهم الجغرافية
  • صلاحيات مشروع: لكل مشروع مدير وفريق مخصص فقط يرون بياناته
  • صلاحيات مبلغ: الفواتير التي تتجاوز مبلغاً معيناً تتطلب موافقة مدير

9 النسخ الاحتياطي

يوفر النظام آليات متعددة للنسخ الاحتياطي لضمان عدم فقدان البيانات.

أنواع النسخ الاحتياطي

النوعالتكرارالتوطينالمدة المحفوظة
ساعي (Incremental)كل ساعةمحلي7 أيام
يومي كامليومياً الساعة 2:00 صمحلي + سحابي30 يوم
أسبوعيكل أحدسحابي3 أشهر
شهريأول كل شهرسحابي + نسخة خارج الموقع12 شهر

آليات النسخ الاحتياطي

# نسخ احتياطي يدوي عبر أمر Django
python manage.py backup

# نسخ احتياطي لقاعدة البيانات فقط
python manage.py backup --database-only

# نسخ احتياطي مع ضغط
python manage.py backup --compress

# استعادة من نسخة احتياطية
python manage.py restore backup_2026_07_21.dump

جدولة النسخ الاحتياطي التلقائي

# على Linux: إضافة جدول زمني (crontab)
# نسخ احتياطي يومي الساعة 2:00 ص
0 2 * * * cd /home/user/accounting_system && venv/bin/python manage.py backup --compress --cloud-upload >> logs/backup.log 2>&1

# نسخ احتياطي كل ساعة
0 * * * * cd /home/user/accounting_system && venv/bin/python manage.py backup --database-only >> logs/backup.log 2>&1
⚠ قواعد النسخ الاحتياطي
  • يجب أن يكون النسخ الاحتياطي في ثلاثة مواقع (محلي + سحابي + خارج الموقع)
  • يجب اختبار الاستعادة شهرياً للتأكد من سلامة النسخ
  • يجب تشفير جميع النسخ الاحتياطية
  • يجب مراجعة سجلات النسخ الاحتياطي يومياً

10.4 إدارة الجلسات (Session Management)

# إعدادات الجلسات في Django
SESSION_ENGINE = "django.contrib.sessions.backends.cache"

SESSION_EXPIRE_AT_BROWSER_CLOSE = True
SESSION_COOKIE_SECURE = True         # HTTPS فقط
SESSION_COOKIE_HTTPONLY = True        # لا يمكن الوصول عبر JavaScript
SESSION_COOKIE_SAMESITE = "Lax"      # حماية من CSRF

# مراقبة الجلسات
SESSION_COOKIE_NAME = "sessionid"
SESSION_COOKIE_DOMAIN = ".company.com"

# تقييد عدد الجلسات النشطة لكل مستخدم
MAX_SESSIONS_PER_USER = 3
