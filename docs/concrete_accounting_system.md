# نظام المحاسبة المتكامل لإنتاج الخرسانة

> اسم الملف المرجعي: `docs/concrete_accounting_system.md`
> ERP v1.0 — الوحدة: `concrete_production` + التكامل المحاسبي

---

## 1. مقدمة

نظام متكامل يربط **الإنتاج الخرساني** بدورة المحاسبة الكاملة: من طلب العميل،
عبر أمر الإنتاج واستهلاك المواد (الأسمنت من الصوامع)، حتى الفاتورة وتكلفة الم³
والقيود المحاسبية التلقائية.

## 2. المكونات (Models)

| النموذج | الوصف |
|---------|-------|
| `ConcreteMixDesign` | تصميم الخلطة (نسبة الأسمنت/الركام/الماء + المضافات). |
| `MixComponent` | مكوّن الخلطة وكميته لكل م³. |
| `CustomerRequest` | طلب العميل للخرسانة (نوع، كمية، تاريخ التسليم). |
| `ProductionOrder` | أمر الإنتاج الناتج من الطلب؛ يولّد فاتورة بيع تلقائياً عند الإكمال. |
| `Truck` | الشاحنة الناقلة وحالتها. |
| `ProductionBatch` | رسبة إنتاج فعلية مرتبطة بأمر الإنتاج. |
| `DeliverySchedule` | جدول تسليم الشاحنات للعميل/المشروع. |
| `ProductionCost` | تفصيل تكلفة الإنتاج (عمالة، تشغيل، مواد). |
| `Silo` | صومعة أسمنت (سعة، رصيد حالي، نوع). |
| `SiloTransaction` | حركة وارد/صادر للأسمنت؛ **تحدّث رصيد الصومعة تلقائياً**. |

## 3. التدفق التشغيلي

```
طلب العميل (CustomerRequest)
   ↓ اعتماد
أمر الإنتاج (ProductionOrder) + تخصيص شاحنات (Truck/DeliverySchedule)
   ↓ إنتاج
رسبة (ProductionBatch) ← استهلاك مكوّنات الخلطة + خصم أسمنت من الصومعة (SiloTransaction)
   ↓ إكمال
فاتورة بيع (SalesInvoice) [تلقائي] + قيد محاسبي (JournalEntry)
   ↓ تقارير
تكلفة الإنتاج لكل م³ (ProductionCost / production_cost_per_m3)
```

## 4. التكامل المحاسبي

- عند إكمال أمر الإنتاج يُنشأ قيد محاسبي عبر `JournalEntryService` يربط
  (إيراد المبيعات / الذمم / تكلفة الإنتاج / مخزون الأسمنت).
- تكلفة الم³ تُحسب من: تكلفة مكوّنات الخلطة + تكلفة الأسمنت المُسحوب من الصومعة
  + عمالة + تشغيل آلات.
- كل ذلك يظهر في التقارير المالية (قائمة الدخل، المركز المالي، متابعة الربحية
  في `reports`).

## 5. شاشات الوحدة

- لوحة الإنتاج `/concrete/` `dashboard`.
- تصاميم الخلطات `mix_design_list` (مع `MixComponent`).
- طلبات العملاء `customer_request_list`.
- أوامر الإنتاج `production_order_list`.
- الصوامع `silo_list` + حركاتها (تحديث تلقائي للرصيد).
- الشاحنات `truck_list`، التسليم `delivery_list`.
- التكلفة `cost_list` وتقرير تكلفة الم³ `production_cost_per_m3`.

## 6. نقاط يجب مراقبتها

- **رصيد الصومعة**: يعتمد على إشارات `SiloTransaction`؛ أي حركة يدوية مباشرة
  على DB دون transaction ستُفقد المزامنة.
- **توليد الفاتورة**: يحدث مرة واحدة عند الإكمال (`test_no_duplicate_invoice`
  يضمن عدم التكرار).
- **تكلفة الم³**: راجع `ProductionCost` قبل الاعتماد عليها في التسعير.
