FIFO Costing — تكلفة المخزون بالوارد أولاً

جديد في الإصدار 2.2: حساب تكلفة المخزون بدقة باستخدام طريقة FIFO (First In, First Out).

كل دفعة واردة تُسجّل كطبقة مستقلة بسعرها. عند البيع، تُخصم الكمية من أقدم طبقة أولاً.

المفهومالشرح
طبقة مخزونكل دفعة واردة تُسجّل كطبقة بسعرها وتاريخها
FIFOعند البيع، تُخصم الكمية من أقدم طبقة أولاً
التكلفةتكلفة البضاعة المباعة = سعر أقدم طبقة
الرصيدرصيد المخزون = مجموع طبقات المخزون المتبقية
يُطبّق تلقائياً عند ترحيل فاتورة مبيعات أو إذن استلام. يناسب صناعة الخرسانة حيث تتغير أسعار المواد الخام باستمرار.

مشاكل التشغيل الأولي والبورت

🔴 المشكلة 1: PowerShell لا يتعرف على start.bat

الخطأ: The term 'start.bat' is not recognized as the name of a cmdlet

السبب: PowerShell لا يجلب الأوامر من المجلد الحالي بشكل افتراضي ( unlike CMD).

الحل:

# الطريقة الصحيحة في PowerShell:
.\start.bat

# أو باستخدام call:
& .\start.bat
في CMD العادي، start.bat يعمل مباشرة. الفرق في سلوك PowerShell.

🔴 المشكلة 2: start.bat لا يعمل بسبب رموز Unicode

الخطأ: 'g' is not recognized أو أخطاء غريبة عند تشغيل start.bat

السبب: ملف الـ batch يحتوي على رموز Unicode (عربي، رسم خانات ░▒▓، سهام ←→) — Windows Batch لا يدعم Unicode ويُعاملها كأوامر.

الحل:

# ❌ لا تستخدم في ملفات .bat:
echo ╔══════════════════════════╗
echo ← اضغط هنا

# ✅ استخدم ASCII بسيط فقط:
echo ======================================
echo   Accounting System - Starting...
echo ======================================
echo.
echo   Press Ctrl+C to stop
القاعدة: ملفات .bat لا تدعم Unicode أبداً. استخدم English فقط أو اكتب تعليقات ASCII.

🔴 المشكلة 3: السيرفر على بورت خاطئ — ERR_CONNECTION_RESET

الخطأ: المتصفح يعرض ERR_CONNECTION_RESET أو This site can't be reached

السبب: تعارض في البورتات — ملف .env يقول بورت واحد، و run_server.py يشغّل على بورت ثابت آخر.

مثال حقيقي:

# في .env:
BIND_PORT=8012

# في network.conf:
PORT = 8012

# لكن run_server.py كان يشغّل على بورت ثابت:
call_command('runserver', '0.0.0.0:8000')  # ← خطأ!

# النتيجة: السيرفر يعمل على 8000 لكن أنت تفتح 8012

الحل:

# 1. تأكد أن run_server.py يقرأ البورت من الإعدادات:
port = os.environ.get('DJANGO_BIND_PORT', '8001')
call_command('runserver', f'0.0.0.0:{port}')

# 2. تأكد network.conf والـ .env عليهما نفس البورت:
# network.conf:
PORT = 8001

# .env:
BIND_PORT=8001

# 3. افحص البورت الفعلي:
netstat -an | findstr "LISTENING"
البورت الصحيح هو ما هو مُعرّف في network.conf و .env معاً — لا تُغيّر أحدهما دون الآخر.

🟡 المشكلة 4: ملف .env لا يُحمّل عند التشغيل

المشكلة: غيرت DJANGO_DEBUG=true في .env لكن النظام لا يزال يتعامل معه كـ false

السبب: run_server.py يستخدم setdefault الذي لا يتجاوز القيم الموجودة مسبقاً في Environment. و Django لا يقرأ ملف .env تلقائياً.

الحل — إحدى الطريقتين:

# الطريقة 1: تثبيت python-dotenv وتحميله في run_server.py
pip install python-dotenv

# في أعلى run_server.py أضف:
from dotenv import load_dotenv
load_dotenv()  # يحمّل .env قبل أي شيء

# الطريقة 2: اضبط المتغيرات يدوياً قبل التشغيل
# في PowerShell:
$env:DJANGO_DEBUG="true"
$env:DJANGO_BIND_PORT="8001"
.\start.bat
الأفضل: استخدم python-dotenv في run_server.py — يضمن أن كل إعدادات .env تُطبّق.

🟡 المشكلة 5: accessing via wrong IP/port from another machine

المشكلة: تفتح http://127.0.0.1:8001 من جهاز آخر فلا يعمل

السبب: 127.0.0.1 يشير للجهاز المحلي فقط — من جهاز آخر استخدم IP الشبكة.

الحل:

# من نفس الجهاز:
http://127.0.0.1:8001

# من جهاز آخر على الشبكة:
http://192.168.1.31:8001

# من الإنترنت (إذا كان external IP مُوجَّه):
http://196.218.24.45:8001
تأكد أن network.conf يحتوي على INTERNAL_IP الصحيح لشبكتك.

🔵 المشكلة 6: السيرفر يعمل لكن الصفحة بيضاء (500 error)

المشكلة: السيرفر يعمل (يظهر في netstat) لكن المتصفح يعرض صفحة بيضاء أو خطأ 500

الأسباب المحتملة:

  • DJANGO_DEBUG=false — Django يخفي تفاصيل الخطأ في الإنتاج
  • خطأ في الهجرات (migrations)
  • ملف .env غير مكتمل (نقص SECRET_KEY مثلاً)

الحل:

# 1.فعّل وضع التصحيح مؤقتاً:
DJANGO_DEBUG=true  # في ملف .env

# 2. شغّل السيرفر يدوياً لرؤية الخطأ:
venv\Scripts\python run_server.py

# 3. الخطأ سيظهر في terminal — صلّحه ثم أعد false

🔵 المشكلة 7: التشغيل من CMD vs PowerShell

المشكلة: نفس الأمر يعمل في CMD لكن يفشل في PowerShell (أو العكس)

السبب: PowerShell و CMD لهما سلوكيات مختلفة:

الإجراءCMDPowerShell
تشغيل ملفstart.bat.\start.bat
ضبط متغير بيئةset DJANGO_DEBUG=true$env:DJANGO_DEBUG="true"
التحقق من متغيرecho %DJANGO_DEBUG%$env:DJANGO_DEBUG
إيقاف عمليةCtrl+CCtrl+C
حذف ملفdel file.txtRemove-Item file.txt
المُوصى به: استخدم CMD للتشغيل العادي — أقل مشاكل مع ملفات .bat

🟡 المشكلة 8: "Site can't be reached" من متصفح آخر على الشبكة

المشكلة: النظام يعمل على الجهاز المحلي لكن لا يصل إليه من أجهزة أخرى

الأسباب:

  1. السيرفر يُسمع على 127.0.0.1 فقط (localhost)
  2. جدار الحماية (Firewall) يمنع البورت
  3. السيرفر يُسمع على 0.0.0.0 لكن على بورت مختلف

الحل:

# 1. تأكد السيرفر على 0.0.0.0 وليس 127.0.0.1:
# run_server.py يجب أن يكون:
call_command('runserver', '0.0.0.0:8001')

# 2. افتح البورت في جدار الحماية (Run as Admin):
netsh advfirewall firewall add rule name="Accounting 8001" dir=in action=allow protocol=TCP localport=8001

# 3. تحقق من السيرفر:
netstat -an | findstr ":8001" | findstr "LISTENING"
# يجب أن يظهر: 0.0.0.0:8001 LISTENING
